Nginx Linux 安装

Feb 22, 2018点击

一、什么是 nginx?

1.1、简介

nginx是一种高性能的HTTP和反向代理服务器 ，同时也是一个代理邮件服务器，我们在nginx上可以发布网站，也可以实现负载均衡的功能 ，还可以作为邮件服务器实现收发邮件等功能 。所谓负载均衡是指当同时有多个用户访问服务器的时候，为了减轻服务器压力，我们需要将用户分别引入各服务器，分担服务器的压力。
nginx 可以实现高并发、部署简单、 内存消耗少、成本低等优点。

1.2、Nginx相对于Apache优点：

  • 高并发响应性能非常好，官方Nginx处理静态文件并发5w/s
  • 反向代理性能非常强。（可用于负载均衡）
  • 内存和cpu占用率低。（为Apache的1/5-1/10）
  • 对后端服务有健康检查功能
  • 支持PHP cgi方式和fastcgi方式
  • 配置代码简洁且容易上手

1.3、Nginx的工作原理：

Nginx由内核和模块组成，其中，内核的设计非常微小和简洁，完成的工作也非常简单，仅仅通过查找配置文件将客户端请求映射到一个location block（location是Nginx配置中的一个指令，用于URL匹配），而在这个location中所配置的每个指令将会启动不同的模块去完成相应的工作。

Nginx的模块从结构上分为核心模块、基础模块和第三方模块：

  • 核心模块：HTTP模块、EVENT模块和MAIL模块
  • 基础模块：HTTP Access模块、HTTP FastCGI模块、HTTP Proxy模块和HTTP Rewrite模块，
  • 第三方模块：HTTP Upstream Request Hash模块、Notice模块和HTTP Access Key模块。

Nginx的高并发得益于其采用了epoll模型，与传统的服务器程序架构不同，epoll是linux内核2.6以后才出现的。Nginx采用epoll模型，异步非阻塞，而apache采用的是select模型：

Select特点：select 选择句柄的时候，是遍历所有句柄，也就是说句柄有事件响应时，select需要遍历所有句柄才能获取到哪些句柄有事件通知，因此效率是非常低。

epoll的特点：epoll对于句柄事件的选择不是遍历的，是事件响应的，就是句柄上事件来就马上选择出来，不需要遍历整个句柄链表，因此效率非常高。

nginx 默认端口号为 80 ,在 Linux 中，默认 0 —— 1024 号端口需要 root用户才能启动，所以需要切换到 root用户，或者更改nginx 默认端口号。

  • nginx 平滑启动
    进入 ${nginx}/sbin目录，执行 ./sbin nginx -s reload ，平滑启动必须是nginx已运行状态
  • nginx 停止
    进入 ${nginx}/sbin目录，执行 ./nginx stop

2、使用 yum 安装

Centos系统默认没有nginx 的 yum 配置，需要自己在 /etc/yum.repos.d目录下创建 nginx.repo文件，内容如下:

参数 描述
name 对repository 的描述，支持 $basearch 这个的变量
baseurl 是服务器设置中最重要的部分，只有设置正确，才能从上面获取软件
gpgcheck 有1和0两个选择，分别代表是否是否进行gpg(GNU Private Guard) 校验，以确定rpm 包的来源是有效和安全的
enable 有 1和0 两个选择

上面的baseurl中，需要将OS修改为你的操作系统名称如rhel 或者 centos,OSRELEASE 修改为6或7，这个要看你的操作系统版本，如我使用的 centos 7，则 baseurl 应该改为 http://nginx.org/packages/centos/7/$basearch/

保存更改后，就可以使用 yum -y install nginx 来安装了，安装后的nginx配置信息如下：
网站文件存放的默认目录 :/usr/share/nginx/html
网站默认站点配置:/etc/nginx/conf.d/default.conf
自定义Nginx站点配置文件存放目录:/etc/nginx/conf.d/
Nginx全局配置:/etc/nginx/nginx.conf

nginx 启动命令:systemctl start nginx
nginx 停止命令:systemctl stop nginx
nginx 重载命令:nginx -s reload
